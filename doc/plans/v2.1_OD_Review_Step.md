# v2.1 OD Review Step — 구현 계획서

> **Codex 교차검토 반영 (2026-03-24)**: Finding 1~4 전체 반영 완료

## 개요

OD ON 캡처 시 YOLO 검출 → Gemini 변환 사이에 **사용자가 검출 결과를 확인/편집할 수 있는 선택적 단계**를 추가한다.

## 현재 흐름 vs 변경 흐름

```
[현재]  영역 드래그 → YOLO 검출 + Gemini 변환 (atomic) → 에디터 삽입

[v2.1]  영역 드래그 → YOLO 검출 → [선택] 검출 결과 오버레이 + 사용자 편집 → Gemini 변환 + figure 후처리 → 에디터 삽입
```

- 기본(default): 리뷰 단계 OFF → 기존과 동일하게 자동 진행
- 사용자 선택: OD Review 토글 ON → 검출 결과 표시 후 확인/편집 → "변환 실행"으로 진행

## 핵심 설계 결정

1. **IPC 분리**: 기존 `capture:analyze` (atomic)를 `capture:detect` (OD only) + `capture:convert` (Gemini + figure post-processing)로 분리
2. **기존 호환성**: `capture:analyze`는 그대로 유지 (리뷰 OFF 시 기존 경로 사용 가능)
3. **리뷰 세션 스냅샷**: detect 시점에 `PendingOdReview` 세션 객체를 생성하여 source context를 immutable하게 고정 (세션 저장 X)
4. **좌표계**: 검출 box_px는 캡처 이미지 픽셀 좌표, 오버레이 표시 시 스케일 변환
5. **변환 전 재정렬**: `convert_regions()` 호출 직전에 편집된 detections를 reading-order로 재정렬
6. **재시도 = convert 재실행**: 변환 실패 시 편집된 detections를 보존하여 convert만 재실행 (detect 재실행 아님)

## 새 타입 정의 (`src/types/index.ts`)

```typescript
export type OdRegionType = 'text' | 'table' | 'figure' | 'formula' | 'abandon'

export interface OdDetection {
  id: string                    // 클라이언트 UUID (편집 추적용)
  label: string                 // YOLO 원본 라벨
  region: OdRegionType          // 매핑된 유형
  score: number                 // 신뢰도 0-1
  box_px: [number, number, number, number]  // [x1, y1, x2, y2] 캡처 이미지 좌표
}

export interface OdDetectionResult {
  detections: OdDetection[]
  imageWidth: number
  imageHeight: number
  error: string | null
}

export interface OdConvertPayload {
  imageBase64: string
  detections: OdDetection[]
  pdfPath?: string
  pageNum?: number
  captureBboxNorm?: number[]
}

/** [F2] 리뷰 세션 — detect 시점에 생성, source context를 immutable하게 고정 */
export interface PendingOdReview {
  imageBase64: string
  imageWidth: number
  imageHeight: number
  pdfPath: string | null
  pageNum: number               // 0-based
  captureBboxNorm: number[]
  blockId: string
  detections: OdDetection[]     // 편집 가능 (mutable copy)
}
```

## 리스크 및 주의사항

1. **좌표 정밀도**: 캡처 이미지 좌표 ↔ 오버레이 표시 좌표 간 스케일 변환 정확도
2. **마우스 이벤트 충돌**: OdReviewOverlay와 AreaCapture의 드래그 핸들러 충돌 → stopPropagation 필수
3. **대량 영역**: YOLO가 20+ 영역 반환 시 UI 혼잡 → z-order 관리 필요
4. **빈 검출 후 편집**: 사용자가 모든 영역 삭제 → 전체 이미지 Gemini fallback으로 처리
5. **abandon 처리**: 사용자가 abandon으로 설정한 영역은 convert 시 필터링
6. **하위 호환**: `capture:analyze` IPC 및 `analyze_capture()` 함수 유지
7. **메모리**: 리뷰 중 PendingOdReview (캡처 base64 수 MB 포함) → 리뷰 종료 시 즉시 해제
8. **세션 미저장**: 리뷰 상태는 transient — 앱 종료 시 소실 (의도적)
9. **[F1] figure 후처리**: `convert_regions()`가 `_replace_figure_markers` + `_get_figure_image` + PDF 300DPI 렌더링을 반드시 포함해야 함
10. **[F2] navigation lock**: 리뷰 중 페이지/줌/툴 변경 시 discard 확인 필수
11. **[F3] 편집 후 재정렬**: convert 직전 reading-order sort 재적용 + 재시도 시 편집본 보존
